# CC Switch v3.16.104

> Key 池可用性与 403 风险控制补丁：当使用统计请求日志捕获到上游 403 时，自动禁用触发请求的密钥，并同步禁用所有应用和供应商中相同的 key；Key 池摘要改为 `总数/可用/禁用`，让禁用 key 与冷却/不稳定 key 的状态更容易区分。

---

## 变更说明

### 403 自动禁用相同 Key

使用统计请求日志写入成功后，如果本次请求的 HTTP 状态码是 403，系统会通过日志中的 `provider_key_id` 找到实际使用的 `key_value`，并禁用所有相同 `key_value` 的 Key 池条目。

这次禁用不限制 `app_type` 或 `provider_id`。因此同一个 key 如果同时配置在 Claude、Codex、Gemini，或同一应用下的多个 provider 中，只要其中一次请求被上游返回 403，所有相同 key 都会被禁用，避免后续调度继续命中已经被拒绝的密钥。

### Key 池摘要改为三段计数

供应商列表和编辑弹窗中的 Key 池摘要从“可用/总数”或“异常”文案调整为固定三段：

```text
Key 总数/可用/禁用
```

例如 `Key 52/40/1` 表示：

- 总共有 52 个 key；
- 当前有 40 个 key 可用；
- 有 1 个 key 被禁用或未启用。

冷却和不稳定状态仍会触发警告色与重置冷却入口，但禁用 key 不再计入“异常”。这样可以区分“用户主动停用 / 403 自动禁用”和“仍在启用但暂时不可用”的 key。
